## <a name="overview"></a><span data-ttu-id="68067-105">Visão geral</span><span class="sxs-lookup"><span data-stu-id="68067-105">Overview</span></span> 
<span data-ttu-id="68067-106">Após publicar um aplicativo de Proxy de Aplicativo, os únicos links que funcionam por padrão no aplicativo são links para destinos contidos na URL raiz publicada.</span><span class="sxs-lookup"><span data-stu-id="68067-106">After publishing an Application Proxy app, the only links that work by default in the application are links to destinations contained within the published root URL.</span></span> <span data-ttu-id="68067-107">Os links nos aplicativos não estão funcionando, a URL interna para o aplicativo provavelmente não inclui todos os destinos de links dentro do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="68067-107">The links within the applications aren’t working, the internal URL for the application probably does not include all the destinations of links within the application.</span></span>

<span data-ttu-id="68067-108">**Por que isso acontece?**</span><span class="sxs-lookup"><span data-stu-id="68067-108">**Why does this happen?**</span></span> <span data-ttu-id="68067-109">Ao clicar em um link em um aplicativo, o Proxy de aplicativo tenta resolver a URL como uma URL interna dentro do mesmo aplicativo ou como uma URL disponível externamente.</span><span class="sxs-lookup"><span data-stu-id="68067-109">When clicking a link in an application, Application Proxy tries to resolve the URL as either an internal URL within the same application, or as an externally available URL.</span></span> <span data-ttu-id="68067-110">Se o link aponta para uma URL interna que não está dentro do mesmo aplicativo, ele não pertence a nenhum um desses buckets e resulta em um erro não encontrado.</span><span class="sxs-lookup"><span data-stu-id="68067-110">If the link points to an internal URL that is not within the same application, it does not belong to either of these buckets and result in a not found error.</span></span>

## <a name="ways-you-can-resolve-broken-links"></a><span data-ttu-id="68067-111">Maneiras de resolver links desfeitos</span><span class="sxs-lookup"><span data-stu-id="68067-111">Ways you can resolve broken links</span></span>

<span data-ttu-id="68067-112">Há três maneiras de resolver esse problema.</span><span class="sxs-lookup"><span data-stu-id="68067-112">There are three ways to resolve this issue.</span></span> <span data-ttu-id="68067-113">As escolhas abaixo estão listadas em crescente complexidade.</span><span class="sxs-lookup"><span data-stu-id="68067-113">The choices below are in listed in increasing complexity.</span></span>

    <span data-ttu-id="68067-116">Se você alterar a URL interna, mas não quiser alterar a página de aterrissagem para usuários, altere a URL da home page para a URL interna publicada anteriormente.</span><span class="sxs-lookup"><span data-stu-id="68067-116">If you change the internal URL but don’t want to change the landing page for users, change the Home page URL to the previously published internal URL.</span></span> <span data-ttu-id="68067-117">Isso pode ser feito indo para "Azure Active Directory" -&gt; Registros de aplicativo -&gt; selecione o aplicativo -&gt; Propriedades.</span><span class="sxs-lookup"><span data-stu-id="68067-117">This can be done by going to “Azure Active Directory” -&gt; App Registrations -&gt; select the application -&gt; Properties.</span></span> <span data-ttu-id="68067-118">Neste guia de propriedades, você verá o campo "URL da Home Page" que pode ser ajustado para a página de aterrissagem desejada.</span><span class="sxs-lookup"><span data-stu-id="68067-118">In this properties tab, you see the field “Home Page URL” which you can adjust to be the desired landing page.</span></span>

    <span data-ttu-id="68067-121">Essa opção garante que os links em seu aplicativo sejam acessíveis externamente através do Proxy de Aplicativo, pois os links dentro do aplicativo para URLs internas também são reconhecidos externamente.</span><span class="sxs-lookup"><span data-stu-id="68067-121">This option ensures that the links in your application are externally accessible through Application Proxy since the links within the application to internal URLs are also recognized externally.</span></span> <span data-ttu-id="68067-122">Observe que todos os links ainda precisam pertencer a um aplicativo publicado.</span><span class="sxs-lookup"><span data-stu-id="68067-122">Note that all links still need to belong to a published application.</span></span> <span data-ttu-id="68067-123">No entanto, com essa opção os links não precisam pertencer ao mesmo aplicativo e podem pertencer a vários aplicativos.</span><span class="sxs-lookup"><span data-stu-id="68067-123">However, with this option the links do not need to belong to the same application and can belong to multiple applications.</span></span>

3.  <span data-ttu-id="68067-124">Se nenhuma dessas opções for viável, você ingressa na visualização para um novo recurso que faz conversão/regravação de URL.</span><span class="sxs-lookup"><span data-stu-id="68067-124">If neither of these options are feasible, you join the preview for a new feature that do URL translation/rewriting.</span></span> <span data-ttu-id="68067-125">Com essa opção, as URLs internas ou os links existentes no corpo HTML de seus aplicativos serão convertidos ou “mapeados” para URLs de Proxy de Aplicativo externas publicadas.</span><span class="sxs-lookup"><span data-stu-id="68067-125">With this option, internal URLs or links that exist in the HTML body of your applications be translated, or “mapped”, to the published external App Proxy URLs.</span></span> <span data-ttu-id="68067-126">Isso só funciona para links em HTML ou CSS, e não irá ajudar se seu link for gerado através de JS.</span><span class="sxs-lookup"><span data-stu-id="68067-126">This only works for links in the HTML or CSS, and this not help if your link is generated through JS.</span></span> 

<span data-ttu-id="68067-127">Como um resultado, é altamente recomendável usar a solução [domínios personalizados](https://docs.microsoft.com/azure/active-directory/active-directory-application-proxy-custom-domains), se possível.</span><span class="sxs-lookup"><span data-stu-id="68067-127">As a result, we strongly recommend using the [custom domains](https://docs.microsoft.com/azure/active-directory/active-directory-application-proxy-custom-domains) solution if possible.</span></span> <span data-ttu-id="68067-128">Se você deseja ingressar na visualização, envie um email <aadapfeedback@microsoft.com> com o applicationId(s).</span><span class="sxs-lookup"><span data-stu-id="68067-128">If you do want to join the preview, email <aadapfeedback@microsoft.com> with the applicationId(s).</span></span>
