<span data-ttu-id="e3a70-106">Em outubro de 2013, o Portal clássico do Azure e as APIs de Gerenciamento de Serviço foram integrados ao Active Directory do Azure para definir as bases visando melhorar a experiência do usuário no gerenciamento do acesso aos recursos do Azure.</span><span class="sxs-lookup"><span data-stu-id="e3a70-106">In October 2013, the Azure classic portal and Service Management APIs were integrated with Azure Active Directory in order to lay the groundwork for improving the user experience for managing access to Azure resources.</span></span> <span data-ttu-id="e3a70-107">O Active Directory do Azure já fornece ótimos recursos como o gerenciamento de usuários, sincronização de diretórios local, autenticação multifator e controle de acesso do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="e3a70-107">Azure Active Directory already provides great capabilities such as user management, on-premises directory sync, multi-factor authentication, and application access control.</span></span> <span data-ttu-id="e3a70-108">Naturalmente, eles também devem ser disponibilizados para gerenciar os recursos do Azure globalmente.</span><span class="sxs-lookup"><span data-stu-id="e3a70-108">Naturally, these should also be made available for managing Azure resources across-the-board.</span></span>

<span data-ttu-id="e3a70-109">O controle de acesso no Azure parte de uma perspectiva de cobrança.</span><span class="sxs-lookup"><span data-stu-id="e3a70-109">Access control in Azure starts from a billing perspective.</span></span> <span data-ttu-id="e3a70-110">O proprietário de uma conta do Azure, acessada pela visitação ao [Centro de Contas do Azure](https://account.windowsazure.com/subscriptions), é o Administrador da Conta (AA).</span><span class="sxs-lookup"><span data-stu-id="e3a70-110">The owner of an Azure account, accessed by visiting the  [Azure Accounts Center](https://account.windowsazure.com/subscriptions), is the Account Administrator (AA).</span></span> <span data-ttu-id="e3a70-111">As assinaturas são um contêiner para cobrança, mas também atuam como um limite de segurança: cada assinatura tem um Administrador de Serviços (SA) que pode adicionar, remover e modificar recursos do Azure nessa assinatura usando o [Portal clássico do Azure](https://manage.windowsazure.com/).</span><span class="sxs-lookup"><span data-stu-id="e3a70-111">Subscriptions are a container for billing, but they also act as a security boundary: each subscription has a Service Administrator (SA) who can add, remove, and modify Azure resources in that subscription by using the [Azure classic portal](https://manage.windowsazure.com/).</span></span> <span data-ttu-id="e3a70-112">O SA padrão de uma nova assinatura é o AA, mas o AA pode alterar o SA no Centro de contas do Azure.</span><span class="sxs-lookup"><span data-stu-id="e3a70-112">The default SA of a new subscription is the AA, but the AA can change the SA in the Azure Accounts Center.</span></span>

<span data-ttu-id="e3a70-114">As assinaturas também têm uma associação com um diretório.</span><span class="sxs-lookup"><span data-stu-id="e3a70-114">Subscriptions also have an association with a directory.</span></span> <span data-ttu-id="e3a70-115">O diretório define um conjunto de usuários.</span><span class="sxs-lookup"><span data-stu-id="e3a70-115">The directory defines a set of users.</span></span> <span data-ttu-id="e3a70-116">Estes podem ser usuários do trabalho ou da escola que criou o diretório ou podem ser usuários externos (ou seja, Contas da Microsoft).</span><span class="sxs-lookup"><span data-stu-id="e3a70-116">These can be users from the work or school that created the directory or they can be external users (that is, Microsoft Accounts).</span></span> <span data-ttu-id="e3a70-117">As assinaturas são acessíveis por um subconjunto desses usuários do diretório que foram atribuídos como um serviço SA (Administrador) ou CA (Coadministrador); a única exceção é que, por motivos de herança, Contas da Microsoft (anteriormente chamadas de Windows Live ID) podem ser atribuídas como SA ou CA sem estarem presentes no diretório.</span><span class="sxs-lookup"><span data-stu-id="e3a70-117">Subscriptions are accessible by a subset of those directory users who have been assigned as either Service Administrator (SA) or Co-Administrator (CA); the only exception is that, for legacy reasons, Microsoft Accounts (formerly Windows Live ID) can be assigned as SA or CA without being present in the directory.</span></span>

<br><br>![Controle de acesso no Azure][2]

<span data-ttu-id="e3a70-119">A funcionalidade no Portal clássico do Azure permite que os SAs que se conectaram usando uma Conta da Microsoft alterem o diretório ao qual uma assinatura está associada usando o comando **Editar Diretório** na página **Assinaturas** em **Configurações**.</span><span class="sxs-lookup"><span data-stu-id="e3a70-119">Functionality within the Azure classic portal enables SAs that are signed in using a Microsoft Account to change the directory that a subscription is associated with by using the **Edit Directory** command on the **Subscriptions** page in **Settings**.</span></span> <span data-ttu-id="e3a70-120">Observe que esta operação tem implicações no controle de acesso da assinatura.</span><span class="sxs-lookup"><span data-stu-id="e3a70-120">Note that this operation has implications on the access control of that subscription.</span></span>

> [!NOTE]
> <span data-ttu-id="e3a70-121">O comando **Editar Diretório** no Portal clássico do Azure não está disponível para os usuários que se conectaram usando uma conta corporativa ou de estudante, pois essas contas podem se conectar apenas ao diretório ao qual pertencem.</span><span class="sxs-lookup"><span data-stu-id="e3a70-121">The **Edit Directory** command in the Azure classic portal is not available to users who are signed in using a work or school account because those accounts can sign in only to the directory to which they belong.</span></span>
> 
> 

<br><br>![Fluxo de Logon de Usuário Simples][3]

<span data-ttu-id="e3a70-123">No caso mais simples, uma organização (como por exemplo, Contoso) fará a cobrança e o controle de acesso no mesmo conjunto de assinaturas.</span><span class="sxs-lookup"><span data-stu-id="e3a70-123">In the simple case, an organization (such as Contoso) will enforce billing and access control across the same set of subscriptions.</span></span> <span data-ttu-id="e3a70-124">Ou seja, o diretório está associado às assinaturas pertencentes a uma única conta do Azure.</span><span class="sxs-lookup"><span data-stu-id="e3a70-124">That is, the directory is associated to subscriptions that are owned by a single Azure Account.</span></span> <span data-ttu-id="e3a70-125">Após o logon bem-sucedido no Portal clássico do Azure, os usuários veem duas coleções de recursos (representadas em laranja na ilustração anterior):</span><span class="sxs-lookup"><span data-stu-id="e3a70-125">Upon successful login to the Azure classic portal, users see two collections of resources (depicted in orange in the previous illustration):</span></span>

* <span data-ttu-id="e3a70-126">Diretórios onde sua conta de usuário existe (originados ou adicionados como uma entidade externa).</span><span class="sxs-lookup"><span data-stu-id="e3a70-126">Directories where their user account exists (sourced or added as a foreign principal).</span></span> <span data-ttu-id="e3a70-127">Observe que o diretório usado para logon não é relevante para esse cálculo, portanto os diretórios serão sempre mostrados independentemente de onde você fez logon.</span><span class="sxs-lookup"><span data-stu-id="e3a70-127">Note that the directory used for login isn’t relevant to this computation, so your directories will always be shown regardless of where you logged in.</span></span>
* <span data-ttu-id="e3a70-128">Recursos que fazem parte das assinaturas que estão associados com o diretório usado para fazer logon E que o usuário pode acessar (seja um SA ou CA).</span><span class="sxs-lookup"><span data-stu-id="e3a70-128">Resources that are part of subscriptions that are associated with the directory used for login AND which the user can access (where they are an SA or CA).</span></span>

<br><br>![Usuário com Várias Assinaturas e Diretórios][4]

<span data-ttu-id="e3a70-130">Os usuários com assinaturas em vários diretórios têm a capacidade de alternar o contexto atual do Portal clássico do Azure usando o filtro de assinatura.</span><span class="sxs-lookup"><span data-stu-id="e3a70-130">Users with subscriptions across multiple directories have the ability to switch the current context of the Azure classic portal by using the subscription filter.</span></span> <span data-ttu-id="e3a70-131">Internamente, isso resulta em um logon separado para um diretório diferente, mas isso é feito diretamente usando logon único (SSO).</span><span class="sxs-lookup"><span data-stu-id="e3a70-131">Under the covers, this results in a separate login to a different directory, but this is accomplished seamlessly using single sign-on (SSO).</span></span>

<span data-ttu-id="e3a70-132">Operações como mover recursos entre as assinaturas podem ser mais difíceis como resultado desta visão única de diretório de assinaturas.</span><span class="sxs-lookup"><span data-stu-id="e3a70-132">Operations such as moving resources between subscriptions can be more difficult as a result of this single directory view of subscriptions.</span></span> <span data-ttu-id="e3a70-133">Para executar a transferência de recursos, talvez seja necessário usar primeiro o comando **Editar Diretório** na página Assinaturas em **Configurações** para associar as assinaturas ao mesmo diretório.</span><span class="sxs-lookup"><span data-stu-id="e3a70-133">To perform the resource transfer, it may be necessary to first use the **Edit Directory** command on the Subscriptions page in **Settings** to associate the subscriptions to the same directory.</span></span>
