## <a name="locks"></a><span data-ttu-id="354cb-141">Bloqueios</span><span class="sxs-lookup"><span data-stu-id="354cb-141">Locks</span></span>
<span data-ttu-id="354cb-142">Nas Coleções Confiáveis, todas as transações implementam um bloqueio de duas fases rigoroso: uma transação não libera os bloqueios que adquiriu até que a transação seja encerrada com uma confirmação ou anulação.</span><span class="sxs-lookup"><span data-stu-id="354cb-142">In Reliable Collections, all transactions implement rigorous two phase locking: a transaction does not release the locks it has acquired until the transaction terminates with either an abort or a commit.</span></span>

<span data-ttu-id="354cb-143">Dicionário Confiável usa bloqueio para todas as operações de entidade única em nível de linha.</span><span class="sxs-lookup"><span data-stu-id="354cb-143">Reliable Dictionary uses row level locking for all single entity operations.</span></span>
<span data-ttu-id="354cb-144">Fila Confiável compensa simultaneidade para propriedade PEPS transacional estrita.</span><span class="sxs-lookup"><span data-stu-id="354cb-144">Reliable Queue trades off concurrency for strict transactional FIFO property.</span></span>
<span data-ttu-id="354cb-145">Fila Confiável usa bloqueios de nível de operação permitindo que uma transação com `TryPeekAsync` e/ou `TryDequeueAsync` e uma transação com `EnqueueAsync` por vez.</span><span class="sxs-lookup"><span data-stu-id="354cb-145">Reliable Queue uses operation level locks allowing one transaction with `TryPeekAsync` and/or `TryDequeueAsync` and one transaction with `EnqueueAsync` at a time.</span></span>
<span data-ttu-id="354cb-146">Observe que, para preservar PEPS, se um `TryPeekAsync` ou `TryDequeueAsync` nunca observarem que a Fila Confiável está vazia, também bloquearão `EnqueueAsync`.</span><span class="sxs-lookup"><span data-stu-id="354cb-146">Note that to preserve FIFO, if a `TryPeekAsync` or `TryDequeueAsync` ever observes that the Reliable Queue is empty, they will also lock `EnqueueAsync`.</span></span>

<span data-ttu-id="354cb-147">Operações de gravação sempre utilizam bloqueios exclusivos.</span><span class="sxs-lookup"><span data-stu-id="354cb-147">Write operations always take Exclusive locks.</span></span>
<span data-ttu-id="354cb-148">Para operações de leitura, o bloqueio depende de alguns fatores.</span><span class="sxs-lookup"><span data-stu-id="354cb-148">For read operations, the locking depends on a couple of factors.</span></span>
