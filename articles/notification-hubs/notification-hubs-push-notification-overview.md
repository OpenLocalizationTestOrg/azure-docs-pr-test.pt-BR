   * <span data-ttu-id="20232-187">Observe que os aplicativos móveis removem imediatamente todas as marcas nos registros para fins de segurança.</span><span class="sxs-lookup"><span data-stu-id="20232-187">Note that Mobile Apps strips away all tags on registrations for security purposes.</span></span> <span data-ttu-id="20232-188">Trabalhar com Hubs de notificação do seu back-end diretamente para associar marcas a dispositivos.</span><span class="sxs-lookup"><span data-stu-id="20232-188">Work with Notification Hubs from your backend directly to associate tags with devices.</span></span>
3. <span data-ttu-id="20232-189">Enviar notificações do seu back-end de aplicativo com Hubs de notificação</span><span class="sxs-lookup"><span data-stu-id="20232-189">Send notifications from your app backend with Notification Hubs</span></span>

<span data-ttu-id="20232-190">Aqui estão alguns recursos convenientes para desenvolvedores com essa integração:</span><span class="sxs-lookup"><span data-stu-id="20232-190">Here are some conveniences brought to developers with this integration:</span></span>

* <span data-ttu-id="20232-191">**SDKs do cliente de Aplicativos Móveis**: estes SDKs de várias plataformas fornecem APIs simples para registro e se comunicam com o hub de notificação vinculado automaticamente com o aplicativo móvel.</span><span class="sxs-lookup"><span data-stu-id="20232-191">**Mobile Apps Client SDKs**: These multi-platform SDKs provide simple APIs for registration and talk to the notification hub linked up with the mobile app automatically.</span></span> <span data-ttu-id="20232-192">Os desenvolvedores não precisam se aprofundar nas credenciais de Hubs de Notificação e trabalhar com um serviço adicional.</span><span class="sxs-lookup"><span data-stu-id="20232-192">Developers do not need to dig through Notification Hubs credentials and work with an additional service.</span></span>

  * <span data-ttu-id="20232-193">*Notificações por push para um usuário*: os SDKs marcam automaticamente o dispositivo fornecido com a ID de usuário autenticada de Aplicativos Móveis para habilitar o envio de notificações por push para o cenário do usuário.</span><span class="sxs-lookup"><span data-stu-id="20232-193">*Push to user*: The SDKs automatically tag the given device with Mobile Apps authenticated User ID to enable push to user scenario.</span></span>
  * <span data-ttu-id="20232-194">*Notificações por push para um dispositivo*: os SDKs usam automaticamente a ID de instalação de Aplicativos Móveis como GUID para se registrar com os Hubs de Notificação, poupando os desenvolvedores da dificuldade de manter vários GUIDs de serviço.</span><span class="sxs-lookup"><span data-stu-id="20232-194">*Push to device*: The SDKs automatically use the Mobile Apps Installation ID as GUID to register with Notification Hubs, saving developers the trouble of maintaining multiple service GUIDs.</span></span>
* <span data-ttu-id="20232-195">**Modelo de instalação**: os Aplicativos móveis funcionam com o modelo de push mais recente dos Hubs de Notificação para representar todas as propriedades de push associadas a um dispositivo em uma instalação de JSON que se alinha com os Serviços de Notificação por Push e é fácil de usar.</span><span class="sxs-lookup"><span data-stu-id="20232-195">**Installation model**: Mobile Apps works with Notification Hubs' latest push model to represent all push properties associated with a device in a JSON Installation that aligns with Push Notification Services and is easy to use.</span></span>
* <span data-ttu-id="20232-196">**Flexibilidade**: os desenvolvedores sempre podem optar por trabalhar com os Hubs de Notificação diretamente até mesmo com a integração em vigor.</span><span class="sxs-lookup"><span data-stu-id="20232-196">**Flexibility**: Developers can always choose to work with Notification Hubs directly even with the integration in place.</span></span>
* <span data-ttu-id="20232-197">**Experiência integrada no [portal do Azure]**: o Push como uma funcionalidade é representado visualmente em Aplicativos Móveis e os desenvolvedores podem facilmente trabalhar com o hub de notificação associado por meio de Aplicativos Móveis.</span><span class="sxs-lookup"><span data-stu-id="20232-197">**Integrated experience in [Azure portal]**: Push as a capability is represented visually in Mobile Apps and developers can easily work with the associated notification hub through Mobile Apps.</span></span>
