## <a name="partitions-in-azure-storage"></a><span data-ttu-id="9add7-125">Partições no Armazenamento do Azure</span><span class="sxs-lookup"><span data-stu-id="9add7-125">Partitions in Azure Storage</span></span>
<span data-ttu-id="9add7-126">Cada objeto com dados armazenado no Armazenamento do Azure (blobs, mensagens, entidades e arquivos) pertence a uma partição e é identificado por uma chave de partição.</span><span class="sxs-lookup"><span data-stu-id="9add7-126">Every object that holds data that is stored in Azure Storage (blobs, messages, entities, and files) belongs to a partition, and is identified by a partition key.</span></span> <span data-ttu-id="9add7-127">A partição determina como o Armazenamento do Azure equilibra as cargas de blobs, mensagens, entidades e arquivos em servidores a fim de atender às necessidades de tráfego desses objetos.</span><span class="sxs-lookup"><span data-stu-id="9add7-127">The partition determines how Azure Storage load balances blobs, messages, entities, and files across servers to meet the traffic needs of those objects.</span></span> <span data-ttu-id="9add7-128">A chave de partição é exclusiva e é usada para localizar um blob, mensagem ou entidade.</span><span class="sxs-lookup"><span data-stu-id="9add7-128">The partition key is unique and is used to locate a blob, message, or entity.</span></span>

<span data-ttu-id="9add7-129">A tabela acima em [Metas de escalabilidade para contas de armazenamento padrão](#standard-storage-accounts) lista as metas de desempenho para uma única partição para cada serviço.</span><span class="sxs-lookup"><span data-stu-id="9add7-129">The table shown above in [Scalability Targets for Standard Storage Accounts](#standard-storage-accounts) lists the performance targets for a single partition for each service.</span></span>

<span data-ttu-id="9add7-130">As partições afetam o balanceamento de carga e a escalabilidade de cada um dos serviços de armazenamento das seguintes formas:</span><span class="sxs-lookup"><span data-stu-id="9add7-130">Partitions affect load balancing and scalability for each of the storage services in the following ways:</span></span>

* <span data-ttu-id="9add7-131">**Blobs**: a chave de partição de um blob é o nome da conta + nome do contêiner + nome do blob.</span><span class="sxs-lookup"><span data-stu-id="9add7-131">**Blobs**: The partition key for a blob is account name + container name + blob name.</span></span> <span data-ttu-id="9add7-132">Isso significa que cada blob poderá ter sua própria partição se a carga no blob exigir isso.</span><span class="sxs-lookup"><span data-stu-id="9add7-132">This means that each blob can have its own partition if load on the blob demands it.</span></span> <span data-ttu-id="9add7-133">BLOBs podem ser distribuídos em vários servidores para escalar horizontalmente o acesso a eles, mas um único blob só pode ser atendido por um único servidor.</span><span class="sxs-lookup"><span data-stu-id="9add7-133">Blobs can be distributed across many servers in order to scale out access to them, but a single blob can only be served by a single server.</span></span> <span data-ttu-id="9add7-134">Embora os blobs possam ser agrupados logicamente em contêineres de blob, o particionamento não é afetado de forma alguma por esse agrupamento.</span><span class="sxs-lookup"><span data-stu-id="9add7-134">While blobs can be logically grouped in blob containers, there are no partitioning implications from this grouping.</span></span>
* <span data-ttu-id="9add7-135">**Arquivos**: A chave de partição para um arquivo é o nome da conta + nome do compartilhamento do arquivos.</span><span class="sxs-lookup"><span data-stu-id="9add7-135">**Files**: The partition key for a file is account name + file share name.</span></span> <span data-ttu-id="9add7-136">Isso significa que todos os arquivos em um compartilhamento de arquivos também estão em uma única partição.</span><span class="sxs-lookup"><span data-stu-id="9add7-136">This means all files in a file share are also in a single partition.</span></span>
* <span data-ttu-id="9add7-137">**Mensagens**: a chave de partição de uma mensagem é o nome da conta + nome da fila, portanto, todas mensagens em uma fila são agrupadas em uma única partição e são atendidas por um único servidor.</span><span class="sxs-lookup"><span data-stu-id="9add7-137">**Messages**: The partition key for a message is the account name + queue name, so all messages in a queue are grouped into a single partition and are served by a single server.</span></span> <span data-ttu-id="9add7-138">Filas diferentes podem ser processadas por servidores diferentes a fim de equilibrar a carga, não importa a quantidade de filas que uma conta de armazenamento tenha.</span><span class="sxs-lookup"><span data-stu-id="9add7-138">Different queues may be processed by different servers to balance the load for however many queues a storage account may have.</span></span>
* <span data-ttu-id="9add7-139">**Entidades**: a chave de partição de uma entidade é nome da conta + nome da tabela + chave de partição, sendo que a chave de partição é o valor da propriedade obrigatória **PartitionKey** definida pelo usuário para a entidade.</span><span class="sxs-lookup"><span data-stu-id="9add7-139">**Entities**: The partition key for an entity is account name + table name + partition key, where the partition key is the value of the required user-defined **PartitionKey** property for the entity.</span></span> <span data-ttu-id="9add7-140">Todas as entidades com o mesmo valor de chave de partição são agrupadas na mesma partição e são atendidas pelo mesmo servidor de partição.</span><span class="sxs-lookup"><span data-stu-id="9add7-140">All entities with the same partition key value are grouped into the same partition and are served by the same partition server.</span></span> <span data-ttu-id="9add7-141">É importante entender isso ao projetar seu aplicativo.</span><span class="sxs-lookup"><span data-stu-id="9add7-141">This is an important point to understand in designing your application.</span></span> <span data-ttu-id="9add7-142">Seu aplicativo deve equilibrar os benefícios de escalabilidade da propagação de entidades por várias partições com as vantagens de acesso de dados do agrupamento de entidades em uma única partição.</span><span class="sxs-lookup"><span data-stu-id="9add7-142">Your application should balance the scalability benefits of spreading entities across multiple partitions with the data access advantages of grouping entities in a single partition.</span></span>  

<span data-ttu-id="9add7-143">Uma vantagem importante de agrupar em uma única partição um conjunto de entidades contidas em uma tabela é a possibilidade de executar operações de lote atômicas nas entidades da mesma partição, já que uma partição existe em um único servidor.</span><span class="sxs-lookup"><span data-stu-id="9add7-143">A key advantage to grouping a set of entities in a table into a single partition is that it's possible to perform atomic batch operations across entities in the same partition, since a partition exists on a single server.</span></span> <span data-ttu-id="9add7-144">Portanto, se você quiser executar operações em lote em um grupo de entidades, considere agrupar essas entidades com a mesma chave de partição.</span><span class="sxs-lookup"><span data-stu-id="9add7-144">Therefore, if you wish to perform batch operations on a group of entities, consider grouping them with the same partition key.</span></span> 

<span data-ttu-id="9add7-145">Por outro lado, entidades que estão na mesma tabela mas têm chaves de partição diferentes podem ter seu balanceamento de carga realizado em servidores diferentes, possibilitando maior escalabilidade.</span><span class="sxs-lookup"><span data-stu-id="9add7-145">On the other hand, entities that are in the same table but have different partition keys can be load balanced across different servers, making it possible to have greater scalability.</span></span>

<span data-ttu-id="9add7-146">Recomendações detalhadas para a criação de estratégias de particionamento de tabelas podem ser encontradas [aqui](https://msdn.microsoft.com/library/azure/hh508997.aspx).</span><span class="sxs-lookup"><span data-stu-id="9add7-146">Detailed recommendations for designing partitioning strategy for tables can be found [here](https://msdn.microsoft.com/library/azure/hh508997.aspx).</span></span>
