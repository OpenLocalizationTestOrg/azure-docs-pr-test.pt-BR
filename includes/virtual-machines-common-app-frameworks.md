## <a name="azure-cli-20"></a>CLI do Azure 2.0
Estas etapas exigem Olá CLI do Azure versão 2.0.4 ou posterior. Executar `az --version` toofind versão de saudação. Se você precisar tooupgrade, consulte [instalar o Azure CLI 2.0]( /cli/azure/install-azure-cli).

Primeiro, crie um grupo de recursos com [az group create](/cli/azure/group#create). Em seguida, crie a sua implantação a partir de um modelo com [az group deployment create](/cli/azure/group/deployment#create). Fornecer modelo de tooyour URI completo Olá Olá `--template-uri` parâmetro e qualquer modelo os parâmetros necessários na `--parameters` da seguinte maneira:

```azurecli
az group create --name myResourceGroup --location eastus
az group deployment create --resource-group myResourceGroup \
    --template-uri https://raw.githubusercontent.com/azure/azure-quickstart-templates/master/{template-name}/azuredeploy.json \
    --parameters '{"sshKeyData": {"value": "ssh-rsa AAAAB3N{snip}B9eIgoZ"}}'
```

Dependendo do modelo de hello, ele pode levar algum tempo para recursos de saudação toodeploy do Azure.

Primeiro, crie um grupo de recursos com [New-AzureRmResourceGroup](/powershell/module/azurerm.resources/new-azurermresourcegroup). Em seguida, crie a sua implantação a partir de um modelo com [New-AzureRmResourceGroupDeployment](/powershell/module/azurerm.resources/new-azurermresourcegroupdeployment). Fornecer modelo de tooyour URI completo Olá Olá `-TemplateURI` parâmetro da seguinte maneira:

```powershell

New-AzureRmResourceGroupDeployment -ResourceGroupName "myResourceGroup" `
     -TemplateUri "https://raw.githubusercontent.com/azure/azure-quickstart-templates/master/{template-name}/azuredeploy.json"
```

Quando você executa Olá [AzureRmResourceGroupDeployment novo](/powershell/module/azurerm.resources/new-azurermresourcegroupdeployment) de comando, pode ser valores de tooenter solicitadas para parâmetros de saudação no modelo de saudação.
